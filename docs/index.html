🔵

Unsupported Configs

Expected

20 configurations utilize hardware components that are intentionally out of KACE's scope (e.g. RGB/Neopixel chains, ADXL345 accelerometers, or SX1509 GPIO expanders).

KACE identifies these features and aborts generation gracefully for those sections rather than throwing errors, preserving complete safety.

⚠️

Delta Geometry Warnings

Safe Guard

10 configurations parse perfectly but trigger warnings due to Klipper's official source files shipping with placeholder TODO pins (especially Delta printers which require manual physical measurements).

KACE's compiler gates safety-check these pins and prompt the developer during the interactive CLI wizard.

Full sweep conducted on KACE v0.9.0 against Klipper master config branch.

🗺️ Development Roadmap

Phased architecture improvements — each phase independently releasable

✓

Phase 1 — Bootstrap Optimizer

Completed
  • Shallow clone (--depth 1) in install.sh
  • Sparse checkout — docs/ and tests/ excluded from Pi install
  • Smart upgrade detection — skip reinstall if already current
  • Lazy paramiko install — SSH support on-demand only
✓

Phase 2 — Hardware Database

Completed
  • MCU search terms, BLTouch pins, thermistor presets → boards.yaml
  • Full mcu_firmware section — data-driven firmware derivation
  • Precedence validation — warns on incorrect pattern order
  • Hardcoded fallback — zero regression risk if YAML missing
✓

Phase 3 — Test Infrastructure

Completed
  • Single-command test runner (python3 tests/run_tests.py)
  • Unit, regression, snapshot, and sweep test categories
  • 7 golden config fixtures — all supported board families
  • GitHub Actions CI pipeline — runs on every push
4

Phase 4 — Module Manifest

No Risk
  • kace_manifest.json — formal module and file list definition
  • Sparse checkout paths driven by manifest (not hardcoded in install.sh)
  • Subcommand routing prep — kace install <module>
  • Plugin-ready architecture for future community modules
5

Phase 5 — Enhanced Caching

Low Risk
  • Hash-based cache invalidation for board configs
  • kace cache clear subcommand
  • Firmware result caching — skip recompile if inputs unchanged
  • Offline-first resilience improvements
